# Skirk Desktop App

This is the Windows-first portable Skirk client. It imports the same one-line
`skirk:` profiles generated by `skirk setup init` and starts the Go Skirk
client sidecar.

Stack:

- Tauri 2
- React
- Go `skirk` sidecar

What it does:

- imports a one-line `skirk:` config or generated `client.json`;
- stores profiles in app-local or portable data;
- gives each imported profile a stable local client identity, so the same
  copied profile can run on multiple devices at the same time;
- starts/stops the Go Skirk SOCKS and HTTP proxy client;
- supports proxy, Windows system proxy, and Windows VPN modes;
- shows proxy addresses, process status, and logs;
- can bind local proxy listeners to `0.0.0.0` for LAN proxy sharing;
- supports a portable Windows folder layout.

It does not require Google login or `gcloud` on the client machine.

## Development

Build the Skirk sidecar first:

```bash
make build
make build-windows
```

Install frontend dependencies:

```bash
cd clients/desktop
npm install
```

Run the app:

```bash
npm run tauri dev
```

## Portable Windows Layout

Ship a zip/folder like:

```text
Skirk.exe
START_HERE.txt
skirk-portable
portable-data/
sidecars/windows/skirk-sidecar.exe
sidecars/windows/skirk-tunnel.exe
```

Portable mode activates when `portable-data/` or `skirk-portable` exists beside the app executable.
Users should open `Skirk.exe`. The sidecar binaries are internal runtimes that
the desktop app starts and stops.

The app stores:

- `portable-data/config/profiles.json`
- `portable-data/config/settings.json`
- imported profile configs as `portable-data/config/profile-*.skirk` or `profile-*.json`
- logs under `portable-data/logs/`

## Production Notes

System proxy mode changes the current user's Windows HTTP/HTTPS proxy settings,
stores a backup first, and restores that backup on disconnect or app restart.

VPN mode runs the bundled `skirk-tunnel.exe` sidecar, which is sing-box staged as
a separate TUN runtime in release builds. Creating a Windows TUN adapter requires
Administrator privileges.

For command-line testing without the desktop app:

```powershell
.\skirk-windows-amd64.exe serve-client --config .\client.skirk --listen 127.0.0.1:18080 --http-proxy-listen 127.0.0.1:18081
```

Add `--client-id my-windows-pc` for a stable CLI identity. The desktop app
passes its saved profile identity automatically.
